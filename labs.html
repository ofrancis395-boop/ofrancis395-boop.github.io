Overview

This page documents my hands-on Active Directory Domain Services (AD DS) lab, built on Windows Server and Windows 11. The goal of this project was to design and deploy a realistic enterprise environment, including domain creation, OU structure, user management, delegated permissions, and Group Policy verification.

Each step includes explanations and screenshots demonstrating the configuration process. Image placeholders are included below — I will replace them with my actual screenshots.

Step 1 — Windows Server Installation

Installed Windows Server and prepared the environment for Active Directory Domain Services.

Step 2 — Configure Static IP Address

Assigned a static IPv4 address to ensure stable DNS and domain communication.

Step 3 — Install Active Directory Domain Services (AD DS)

Installed the AD DS role and promoted the server to a Domain Controller.

Step 4 — DNS Configuration

Verified DNS installation and ensured the domain controller was authoritative for the new domain.

Step 5 — Create Delegated Domain Join Account

Created a limited-permission account with rights to join computers to the domain, following least-privilege best practices.

Step 6 — Join Windows 11 to the Domain

Joined the Windows 11 client to the domain using the delegated account.

Step 7 — Move Computer to Correct OU

Moved the Windows 11 computer object from the default Computers container into a dedicated Computers OU to support Group Policy application.

Step 8 — Create First Real Domain User

Created a standard user account inside the Staff OU to simulate a real employee identity.

Step 9 — Log Into Windows 11 as Domain User

Logged into the Windows 11 workstation using the new domain user account to verify authentication and profile creation.

Step 10 — Verify Group Policy Processing

Ran gpupdate /force and gpresult /r to confirm that Group Policy was applying correctly.
